Rendering engine

El trabajo de un engine de renderizado es pintar el texto de html en la pantalla.

  • Internet explorer => Trident
  • Firefox => Gecko
  • Safari => Webkit
  • Chrome => Blink

¿Qué ocurre al visitar http://google.com?

  1. Se obtiene el recurso recursos (HTTP GET)
  2. Parseo del HTML
  3. Creación del arbol del DOM
  4. Creación del arbol para renderizado
  5. Layout del arbol para renderizado
  6. Pintar

HTML Parser

  • Input: HTML / Output: parse tree (DOM)
  • HTML no puede ser parseado por un parser común
    1. HTML, por naturaleza, es muy permisivo
    2. Los navegadores tienen, por tradición una gran tolerancia a errores
    3. El código de HTML puede ser modificado (document.write())

Render tree

Contiene la información visual que el web browser necesita para dibujar la página

  • Solo elementos visuales estan en el arbol de renderizado
  • En el arbol de renderizado los nodos corresponden a cajas del box-model
  • Para este punto el browser necesita la información de estilo (Style Computation)
  • CSSOM (CSS Object Model)
  • Los elementos position: fixed y position: absolute no están se agregan al arbol si no hasta el final

Layout

Proceso de agregar dimenciones a los elementos del render tree (ancho, alto)

  • Para el alto, se usa un proceso recursivo en el que cada padre pregunta el ancho a sus hijos y luego determina su alto.
  • Para el ancho, el padre primero calcula su ancho y se le dice a los hijos que se organizen con este ancho

Painting

Cada nodo del arbol de renderizado se pinta a si mismo.

  • z-index: stacking context

re-layout + re-painting

El web browser va a hacer solo el mínimo esfuerzo necesario

  • Al cambiar solo el color de un elemento, solo necesita repintar el elemento
  • Al cambiar el tamaño de un elemento, solo necesita recalcular ese elemento y el de sus hijos
  • Los cambios se van a memoria y luego se ejecutan en batch

¿Cómo puedo hacer un mejor código?

  • Aplicar cambios consecutivos a un elemento no visible (los elementos no visibles estan fuera del arbol)
  • Cambiar una clase en vez de cambiar estilos independientes (solo se hace un re-layout)
  • Mientras más abajo en el arbol se hacen los cambios, tienen un menor impacto
  • Compartir estilos (los hijos comparten estilos del padre)
  • Los rendering engines son single threaded
  • Las tags de script se ejecutan sincronicamente (para que puedan utilizar document.write) y deben colocarse al final del body
  • Las tags de script pueden tener un atributo defer, con el que el browser puede decidir si renderizarla al final del DOM
  • Scripts con defer o async no puden utilizar document.write (o innerHTML)
  • Agregar una tag de script con append la ejecuta inmediatamente
  • Código de script dentro de eventos se ejecuta cuando ocurre el evento
