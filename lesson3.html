Frameworks?

Existen muchos frameworks de CSS. Existen para abstraer conceptos reduntantes y para ahorrar tiempo a los desarrolladores. Se utilizan mucho en el mundo real, y definitivamente vale la pena que las aprendan, pero sólo despues de que entiendan las bases y los problemas que estas solucionan.

Estructura vs presentación

  • em y strong aparecen para reemplazar tags i y b
  • HTML se encarga de la estructura
  • CSS se encarga de la presentación
  • Un elemento de html no quiere decir nada sobre la presentación, lo que estamos viendo son estilos por defecto del navegador.

CSS

Cascading Style Sheets

  • Se encarga de la presentación del contenido
  • Se puede utilizar para estilizar cualquier documento XML (incluyendo HTML y SVG)
  • Fue diseñado para separar la presentación y el contenido

Selector

index.html


              <h1>Hello World</h1>

index.css


              h1 {
                color: Maroon;
              }

Class Selector

index.html


              <h1 class="title">Hello World</h1>

index.css


              h1.title {
                color: Maroon;
              }

ID Selector

index.html


              <h1 class="title" id="num4">Hello World</h1>

index.css


              h1.title#num4 {
                color: Maroon;
              }

Multiple Selector

index.html


              <h1 class="title" id="num4">Hello</h1>
              <h1 class="sub" id="num3">World</h1>

index.css


              .title, .sub {
                color: Maroon;
              }
