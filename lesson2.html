Sub dominio

  • Los subdominios están en el zone file del servidor (en posesión del nameserver)
  • Es una subdivisión de un dominio
  • No necesita estar en el mismo servidor que el dominio
  • Cuando se creo el http, habian varias aplicaciones y se utilizaban los subdominios para organizarlas, por ejemplo ftp.example.com, smtp.example.com, www.example.com

Resumen de DNS

  1. Revisión local del dominio (/etc/hosts o cache)
  2. Solicitúd al servidor de DNS (8.8.8.8, 8.8.4.4)
  3. El servidor de DNS revisa si ya conoce el TLD, si no, se le solicita al root server
  4. Se le solicita al TLD (.com) la IP del nameserver
  5. Se le solicita al nameserver la dirección del servidor
  6. Se le solicita al servidor web el recurso
