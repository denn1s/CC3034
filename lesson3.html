Frameworks?

Existen muchos frameworks. Existen para abstraer conceptos reduntantes y para ahorrar tiempo a los desarrolladores. Se utilizan mucho en el mundo real, y definitivamente vale la pena que las aprendan, pero sólo despues de que entiendan las bases y los problemas que estas solucionan.

Estructura vs presentación

  • em y strong aparecen para reemplazar tags i y b
  • HTML se encarga de la estructura
  • CSS se encarga de la presentación
  • Un elemento de html no quiere decir nada sobre la presentación, lo que estamos viendo son estilos por defecto del navegador.

Imágenes

El contenido de la imágen está definido afuera de la página web que se está pintando

Formatos de imágen

  • Regularmente usamos imágenes JPG, GIF, PNG o SVG

JPG

Son buenos para manejar imágenes con grandes cantidades de colores. No permiten pixeles transparentes.

GIF

Son buenos para animaciones simples, pero tienen una paleta muy limitada de colores. Permiten pixeles transparentes.

PNG

Es una buena opción de propósito general. Permiten pixeles transparentes.

SVG

Son vectoriales. Pueden ser escaladas infinitamente sin perder nada de definición. Permiten pixeles transparentes.

CSS

Cascading Style Sheets

  • Se encarga de la presentación del contenido
  • Esto quiere decir que se encarga de cómo se va a mostrar el contenido al usuario en un web browser
  • Se puede utilizar para estilizar cualquier documento XML (incluyendo HTML y SVG)
  • Fue diseñado para separar la presentación y el contenido

Conectando CSS a HTML


                        <head>
                            <meta charset='UTF-8' />
                            <title>Hello with CSS</title>
                            <link rel='stylesheet' href='styles.css' />
                        </head>
