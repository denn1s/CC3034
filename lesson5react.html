¿Por qué utilizar componentes?

¿Por qué no?

  • ¿Qué desventajas tiene este enfoque?

Trabajar con componentes permite que las tecnologias web se alinien con el Principio de Responsabilidad única. Esto nos permite organizar nuestro código y poder debuggear (y colaborar).

  • One component should focus on one section of the UI.
  • One component shouldn’t access another’s html directly.
  • The component encapsulates its html, css and javascript.
  • Each component has some way to interact with others.

React

Actualmente tenemos dos problemas:

  1. ¿Cómo podemos cambiar el estado de los componentes?
  2. ¿Cómo podemos simplificar la sintaxis?

React es una librería (no un framework!) para crear interfaces de usuario de manera eficiente utilizando código declarativo.

¿Qué es código declarativo?

El código declarativo describe qué y no cómo. El código imperativo describe cómo.

En react, utilizamos código declarativo (JSX) para crear componentes. Los componentes son bloques de la interfaz de usuario que actuan independientemente y que se comunican entre sí utilizando propiedades. Cáda componente tiene un estado que puede cambiar.

Los componentes están organizados en una herarquía en la que la informacón fluye en una sola dirección a traves de las propiedades.

Para poder actualizar los componentes cuando cambia el estado, react crea una copia del DOM, llamada Virtual DOM en la que aplica todos los cambios cada vez que el estado cambia. Despues compara el DOM virtual con el DOM real y ejecuta únicamente las diferencias.
